The performance was abysmal either way... 100% cpu util on both cpu's, 100% on gpu, it was below 1tk/s "Hello" response took like a minute (less but felt like it.) and model in itself felt stupid, it failed basic reasoning test that i typically do - it was also preaching Chinese propaganda, or when you ask it to list top 10 oldest constitutions it would list china as 2nd oldest after Babylon in 1930's(i presume) then US, and it wasn't able to sort by date - it still put china on either 1st or 2nd place above US - and as stated you couldn't reason with it at all...
On the other hand, llama3.3 70B model was much better, and much faster and you could reason with it. It felt more real closer to human if you talked with someone rather than what you get with deepseek.


for those that want to run with pooled memory etc, should look here:

llama.cpp/examples/server at master · ggml-org/llama.cpp

LLM inference in C/C++. Contribute to ggml-org/llama.cpp development by creating an account on GitHub.
github.com
--gpu-layers *number of layers of model matrix that will be stored in gpu
--split-mode *how it should split the model matrix by layers or rows between compute devices *rows seem better.
--tensor-split *load by gpu by id %
--main-gpu *should be your fastest gpu

with those settings if used correctly one can pool of many gpu's, system memory, cpu's and so on - doesn't mean it will produce great performance results but it may allow you to actually run model which you wouldn't normally be able to open.
